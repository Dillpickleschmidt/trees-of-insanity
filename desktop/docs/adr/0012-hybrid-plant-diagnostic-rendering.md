# Use hybrid rendering for Plant diagnostics

Render Plant-workspace spheres, flow paths, and terminal markers through the native depth-aware GPU overlay, while rendering module diagnostic labels as Solid DOM elements for normal typography, formatting, and theming. The GPU overlay is a deep graphics module: its small interface accepts generic batched world lines, branch-surface paths, markers, and translucent spheres, with no Plant-workspace concepts. Plant render projection alone translates plant diagnostics into those primitives, assigns stable module colors to bounding spheres, and supplies native-projected label anchors, so other workspaces carry no plant-specific options or logic.

Bounding spheres render as batched analytic impostors: one camera-facing quad per instance and a fragment-stage ray/sphere intersection provide smooth silhouettes without CPU tessellation or a sphere mesh. Back-to-front instance sorting supplies deterministic source-alpha blending, while scene-distance sampling hides sphere surfaces behind opaque geometry. Spheres render before paths and lines so flow and terminal diagnostics stay legible, and they preserve the viewport target's alpha channel. Branch-surface paths store centerline and host radius; the overlay derives their tangent and expands them into camera-facing world-space ribbons matching the current host-segment diameter. Scene-distance occlusion uses a host-scaled tolerance so the supporting branch and nearby joint geometry cannot hide the ribbon, while sufficiently foreground geometry still can. A shader-global time value animates dash phase without CPU geometry updates: accumulated light moves toward the root and vigor moves toward terminals. Both use a blue-cyan-green-yellow-red weight map for their root-normalized flow fraction. Ribbon width encodes branch structure rather than flow intensity and naturally follows zoom and perspective, while labels retain absolute values. Native graphics also projects module-root label anchors into viewport coordinates and sends screen positions and values through a narrow bridge; TypeScript does not duplicate camera math. Labels use a fixed screen-space offset, hide only outside the view or behind the camera, and deliberately perform no overlap avoidance. Terminal markers appear only on mature modules: occupied terminals are filled, unoccupied terminals are hollow, and color distinguishes the main-axis terminal from lateral terminals. This avoids custom GPU font and label-layout systems without compromising depth-aware geometric diagnostics.
