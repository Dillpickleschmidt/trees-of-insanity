# Growth library requirements

## Product

A deterministic C++ library that develops branch module prototypes and complete single-plant architectures from explicit value inputs.

## Requirements

- Pure, deterministic results for equal inputs.
- Module-scale physiological-age growth.
- Plant-scale light, vigor, attachment, shedding, orientation, and senescence.
- All built-in plant type presets remain finite and bounded.
- Z-up coordinates at the public boundary.
- Explicit validation errors for invalid parameters/prototypes.
- Standalone configure, build, test, and install.

## Exclusions

Files, OBJ, JSON, projects, workspaces, cameras, render meshes, USD, ovrtx, CUDA, Vulkan, Qt, browser UI, and persistence.

No compatibility behavior is required before the first released library schema/API.
